================================================================================
MANTRA AI - VERCEL ENVIRONMENT VARIABLES
================================================================================
Configure these in: Vercel Dashboard > Project Settings > Environment Variables
================================================================================

--------------------------------------------------------------------------------
REQUIRED - API Connection
--------------------------------------------------------------------------------

NEXT_PUBLIC_API_URL=https://api.yourdomain.com
# Description: Base URL for the Go Fiber backend API
# Example: https://api.mantra-ai.com
# Source: Cloudflare Tunnel hostname for your API service
# Used by: All API hooks (useAIProvider, useWhatsApp, useInbox, useTenant)

--------------------------------------------------------------------------------
REQUIRED - WebSocket Connection
--------------------------------------------------------------------------------

NEXT_PUBLIC_WS_URL=wss://api.yourdomain.com
# Description: WebSocket URL for real-time connections (auto-derived from API_URL if not set)
# Example: wss://api.mantra-ai.com
# Source: Same as API URL but with wss:// protocol
# Used by: useInboxWebSocket, useQRCodeStream hooks

--------------------------------------------------------------------------------
OPTIONAL - Feature Flags
--------------------------------------------------------------------------------

NEXT_PUBLIC_ENABLE_DEVTOOLS=false
# Description: Enable React Query DevTools in production
# Values: true | false
# Default: false

NEXT_PUBLIC_ENABLE_MOCK_DATA=false
# Description: Use mock data instead of real API calls (for demos)
# Values: true | false
# Default: false

--------------------------------------------------------------------------------
CLOUDFLARE TUNNEL URLS REFERENCE
--------------------------------------------------------------------------------

When setting up Cloudflare Tunnel, you'll configure these hostnames:

1. api.yourdomain.com -> http://api:3001 (Go Fiber API)
2. evolution.yourdomain.com -> http://evolution:8080 (Evolution API - optional)

The NEXT_PUBLIC_API_URL should point to your API tunnel hostname.

--------------------------------------------------------------------------------
VERCEL PROJECT SETTINGS
--------------------------------------------------------------------------------

In addition to environment variables, configure:

1. Framework Preset: Next.js
2. Build Command: next build (default)
3. Output Directory: .next (default)
4. Install Command: pnpm install (if using pnpm)

--------------------------------------------------------------------------------
CORS CONFIGURATION REMINDER
--------------------------------------------------------------------------------

Ensure your Go Fiber backend allows these origins in CORS config:

Production:
- https://your-project.vercel.app
- https://your-custom-domain.com

Preview Deployments:
- https://*.vercel.app (wildcard for preview URLs)

Development:
- http://localhost:3000

Example Go Fiber CORS middleware:

    app.Use(cors.New(cors.Config{
        AllowOrigins:     os.Getenv("CORS_ORIGINS"),
        AllowMethods:     "GET,POST,PUT,PATCH,DELETE,OPTIONS",
        AllowHeaders:     "Origin,Content-Type,Accept,Authorization",
        AllowCredentials: true,
    }))

--------------------------------------------------------------------------------
TROUBLESHOOTING
--------------------------------------------------------------------------------

Issue: "Failed to fetch" errors
- Check NEXT_PUBLIC_API_URL is correct
- Verify Cloudflare Tunnel is running
- Check CORS configuration in backend

Issue: WebSocket connection fails
- Ensure Cloudflare has WebSockets enabled (SSL/TLS > Edge Certificates)
- Check wss:// protocol is used in production
- Verify no firewall blocking WebSocket connections

Issue: 401 Unauthorized errors
- Check JWT token is being sent in Authorization header
- Verify JWT_SECRET matches between frontend auth and backend
